Getting started



First thing you need to download the most recent release.
You can find all official releases at https://github.com/tncrazvan/ElkServer/releases

Once you've downloaded the java archive, you can now start creating your web application.

Http.json

Make a new file, named http.json.
This file will be fed to the web server, it contains basic information about your web server and your web application, like the character set your server should be using, the port, binding address, ecc..

It's time to write down your server details inside http.json. 

{
    "port":80,
    "webRoot": "www/", //your website directory (relative to this json file)
    "charset": "UTF-8",
    "timeout": 5000,
    "bindAddress": "::" // :: means the server will bind on all available addresses
}

Most of the file contains generic information, I'm going to asume that you are familiar with Socket technology and you understand what many of the fields mean.

Web Controllers

Controllers, as in Model-View-Controller (MVC), are the main request managers for you web application.
We haven't wrote any code yet, however, there are a few controllers set by default, such as ControllerNotFound, which triggers when a given resource is not found on the server.

In order for the web server to know where to find these controllers, the http.json file needs to specify the exact package in which they exist.

In this case, we are not importing the java archive in a customized project, we are instead directly using it as a standalone project, so we don't need to define any package for http and web socket controllers, instead the default packages will be used.

Index.html

Make a new directory named www/ and make a new file www/index.html.
This file will be your application entry point, so throw your html template inside it. 




Start the web server




    Everything is ready, you can start your server now.
    Make sure your Java Environment is installed, and run your server.




java -jar ElkServer-1.0.0.jar http.json
